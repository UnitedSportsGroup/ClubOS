META TRACKING REQUIREMENTS (CRITICAL)
We are running Meta ads and must capture accurate conversion data.
Implement BOTH:
A) Meta Pixel client-side
B) Meta Conversions API server-side

Client-side Pixel
- Include pixel base code on public pages
- On landing page view: Track "ViewContent" with content_name=camp name, content_category="Holiday Camp"
- When user clicks book CTA: Track "InitiateCheckout"
- On successful payment (success page): Track "Purchase" with value and currency

Server-side Conversions API
- On checkout session creation: send "InitiateCheckout" event server-side
- On payment success (webhook or success handler): send "Purchase" event server-side
- Use event_id for deduplication: generate a UUID per event and send same id both client and server for the same event where possible
- Capture and store:
  - fbp and fbc cookies from client (read in browser, pass to server)
  - fbclid from URL if present
  - user agent + IP address (from request headers)
  - external_id can be hashed email (SHA256 lowercased trimmed)
- Use Meta Graph API endpoint:
  - /{PIXEL_ID}/events?access_token={TOKEN}
- Provide env vars:
  - META_PIXEL_ID
  - META_ACCESS_TOKEN
  - STRIPE_SECRET_KEY
  - STRIPE_WEBHOOK_SECRET
  - RESEND_API_KEY
  - DATABASE_URL
  - ADMIN_EMAIL + ADMIN_PASSWORD (or NextAuth secret)
